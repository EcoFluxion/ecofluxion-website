## ğŸ“§ Form Integration

The contact and CV forms currently use client-side validation. To make them fully functional, integrate with a backend service:

### Option 1: EmailJS (Recommended - No Backend Required)

1. Sign up at [emailjs.com](https://www.emailjs.com)
2. Get your service ID, template ID, and public key
3. Add EmailJS SDK before `</body>`:

```html
<script src="https://cdn.jsdelivr.net/npm/@emailjs/browser@3/dist/email.min.js"></script>
<script>
    (function() {
        emailjs.init("YOUR_PUBLIC_KEY");
    })();
</script>
```

4. Update form submission in `js/main.js`:

```javascript
emailjs.send("YOUR_SERVICE_ID", "YOUR_TEMPLATE_ID", formData)
    .then(function(response) {
        showFormMessage('success', translations[currentLanguage].contact_success);
    });
```

### Option 2: FormSpree

1. Sign up at [formspree.io](https://formspree.io)
2. Get your form endpoint
3. Update form action or use AJAX:

```javascript
$.ajax({
    url: "https://formspree.io/f/YOUR_FORM_ID",
    method: "POST",
    data: formData,
    dataType: "json"
});
```

### Option 3: Custom PHP Backend

Create `contact-handler.php`:

```php
<?php
header('Content-Type: application/json');

if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $name = $_POST['name'];
    $email = $_POST['email'];
    // ... process form data
    
    mail("info@ecofluxion.com", "Contact Form", $message);
    
    echo json_encode(["success" => true]);
}
?>
```

Update AJAX URL in `js/main.js`.

## ğŸ”§ Troubleshooting

### Common Issues

**Language doesn't change:**
- Check browser console for errors
- Verify `translations.js` is loaded
- Clear localStorage and refresh

**Animations not working:**
- Check if AOS library loaded
- Verify data-aos attributes exist
- Try different browser

**Forms not submitting:**
- Forms currently show success message (demo mode)
- Integrate with backend service (see Form Integration section)

**Particles not showing:**
- Check if particles.js loaded correctly
- Verify #particles-js div exists
- Check browser console for errors

### Debug Mode

Enable debug mode by adding to console:

```javascript
localStorage.setItem('debug', 'true');
```
